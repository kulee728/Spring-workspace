바탕화면에 이미지 저장 폴더 만들기

	사진을 저장하고 불러올 때 sql 컬럼 자료형으로 varchar2를 사용할 수 있고
	이미지 자체를 저장하는 Blob을 사용할 수 있음
	db자체에 이미지를 저장하게 되면 이미지가 훼손될 가능성이 있거나 이미지를 글자로 변환할 때
	변환에 문제가 생길 가능성이 높음
	
	db 자체에 이미지를 저장하게 되면 비용이 많이 들기 때문에 많이 사용하지는 않는 방법
	
	이미지를 저장할 때 sql 자료형(varchar2)으로 이미지 경로를 설정해주고
	설정한 이미지 경로를 가져오는 방식을 진행
	
	이미지가 저장되는 경로
	src/main/resource
				└static
					└image = 홈페이지에서 공개가 가능하고, 공통으로 사용할 수 있는 이미지
	회사에서 지정한 이미지 저장경로 = 회사가 아닌, 고객이 올리거나 회사에서 비공개로 이미지를 가져오고
							 저장하는 경로		
							 
	라이브러리/프레임워크 설정되지 않은 js는 유출 될 가능성이 높다.
	코드 형식이 라이브러리, 프레임워크 형식이라면 외부에서 코드를 유추하기 어렵다.
	
	-이미지 저장 디렉토리가 사라지거나, 이미지 원본이 덮어씌워지는 등 훼손되거나, 이미지 저장에 문자가 생기는 문제를
	예방하기 위해
	1) 이미지를 불러오거나 저장할 경로가 존재하는지 반드시 확인
		> 경로,이미지 존재 X, 불러오기문제 : '이미지를 불러오는데 실패했습니다.'
		> 이미지를 저장하는 폴더가 존재하지 않음 : 
		'이미지 경로 확인 : 원하는 경로가 없으면 경로를 생성' >> 이미지를 저장하기 위해
	2) 이름의 중복
		-> 이미지를 원하는 폴더에 저장할 때, 중복된 이름 파일이 존재하는 경우
		저장하는 시간을 뒤에 붙여 저장한다.
		bag.jpg >> bag20240624_095035
