스프링 부트 프로젝트 실행방법 (서버 배포 방법) 

	src/main/java 하위 패키지에
	[프로젝트파일명]Application.java 파일 (메인클래스)이 자동적으로 생성된다.
	해당 클래스에는 main이 존재한다.
	>> 실행시 Tomcat 서버가 내부 내장되있어 배포를 바로 시작한다.
	브라우저가 알아서 실행되지 않고 주소를 입력하여 수동으로 실행하면 된다.
	http://localhost:[portnum]/주소
	application.properties에서 server.port=8081 로 설정했기 때문에
	:8081 로 접속 가능하다.

프로젝트 구조

  • src/main/java : java 소스코드 모음 디렉토리
  • src/main/resources : java 사용할 설정/자원 파일 모음 디렉토리
    • templates : Java 템플릿 (응답화면)을 모아두는 폴더 (주로 html)
    • static : templates 에 적용될 정적 자원 js, css, 이미지, 동영상, bootstrap 등
    • application.properties : Spring Boot 프로젝트 요소 설정 파일
    • 메이븐과 그래들 maven : Apache 사 에서 개발한 프로젝트 관리/이해도구 주로 pom.xml 파일형태 존재, 추가할 기능을 작성 가능 gradle : groovy와 코틀린을 이용한 빌드 자동화 시스템. build.gradle 파일 형태 존재, 추가할 기능을 작성 가능 groovy는 java에 파이썬,루비와 같은 특징이 포함된 언어이다. 코틀린 >> 웹 앱을 만드는 언어. *가독성, 재사용용이,빌드스크립트의 구조적 단순함, 편리함, 멀티프로젝트, Maven을 완전 지원.
    1. 요청 + 포워드 처리
    2. 이미지 보러가기
    3. 파라미터 전달하기
